2. Faire tourner Docker sous Windows

On fait ici tourner un MySQL et Apache PHP

Installation

  • Installer Docker
  • Dém arrer un conteneur "docker run --rm alpine echo DockerRocks"
  • Puis installer *Docker for Windows Shared Drive

On va utiliser 2 répertoires

  • 1 pour /www
  • 1 pour les données mysql /sqldata
Le mysql est accessible sur localhost:8881 Le site php est accessible sur localhost:8890 Le php est en version 7. Le mysql est en version 5.7

Choix images

Après recherche sur [https://hub.docker.com](https://hub.docker.com) on choisit l'image standard php:7.2-apache ``` docker run --rm -p 8888:80 --name my-apache-php-app -v c:/workspace_wamp/php-sql/codephp:/var/www/html php:7.2-apache ``` Test en ouvrant [http://localhost:8888](http://localhost:8888) dans Chrome. OK, la page **index.php** s'affiche comme attendu. Donc on passe à la configuration fichier.

Configuration de l'image Apache avec docker-compose.yml

version: '3'
services:
  monphp:
    image: php:7.2-apache
    ports:
      - "8888:80"
    volumes:
      - c:/yann/workspace_wamp/php-sql/codephp:/var/www/html
Voilà déjà notre docker Apache.

Test de l'image MySQL en ligne de commande

De même que pour le choix de l'image Apache, on préfère choisir l'image MySQL standard de l'éditeur plutôt qu'une image d'une origine incertaine. 
docker run --rm -p 8881:3306 --name mon-mysql -v
	c:/workspace_leftjoin/php-sql/sqldata:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=toto mysql:5.7
OK, jdbc:mysql://localhost:8881 répond avec le compte *root* password *toto*

Ecriture du docker-compose.yml parent des deux images

Une différence est à noter pour les deux images MySQL et Apache * l'image **mysql:5.7** est déclarée telle quelle dans docker-compose.yml * l'image de **php:7.2-apache** doit être modifié afin de lui ajouter les extensions permettant la connexion MySQL. Alors du coup on crée un Dockerfile qui déclare ces extensions, et docker-compose.yml y fait référence avec la propriété **build:context**
